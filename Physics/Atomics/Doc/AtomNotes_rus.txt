Атомы создаются на базе 7 уровней, содержащих максимальное количество электронов 2,8,18,32,32,32,32.  
Для получения стерео режима используются сферы электронов красного и синего цветов.

Задействованы объекты:
1 Камера (Camera)
1 Свет (Light)
1 Меш (Mesh) для имитации взрыва или расщепления центрального атома.
1 Объект Линии (Lines) для рисования линий между атомами на каждом уровне.
1 Большая центральная сфера красного цвета
1 Большая центральная сфера синего цвета
1 Маленькая красная сфера используется в качестве шаблона для создания каждого из уровней электронов.
1 Маленькая синяя сфера используется в качестве шаблона для создания каждого из уровней электронов.
14 Dummycubes: по 7 для каждого красного и синего уровней. Используется для вращения уровней в отдельных X,Y,Z вращениях.

(Cadencer и Timer, компоненты GLScene и GLSceneViewer)

В объект Mesh загружается сфера из файла sphere.3ds для последующей имитации взрыва...
Каждый Dummycube включает от 2 до 32 сфер, динамически создаваемых как потомки, для каждого уровня электронов. 
Они используют 1 сферу как шаблон (красного или синего цвета).
В программе есть диалог для изменения цвета выбранного уровня, скорости вращения по X,Y,Z, и переключения стиля
отображения линий, связывающих сферы электронов для выбранного уровня. 
Линия всегда получает измененные абсолютные координаты потомков сфер в Dummycube. 
Вращение управляется временем приращения каденсера (Cadencers Delta time), 
переменной скорости и переменной абсолютной скорости, 
тогда как отображение линий поддерживается собственной скоростью (с меньшей частотой чем для вращения).

Форма About содержит мемо поле с подсказками о действии клавиш. 
Есть возможность изменять скорость вращения +- , задействовать процессор Cpu, свойства камеры (фокусное расстояние 
и глубина поля зрения), угол направления камеры, сброс скорости в значение по умолчанию и разрешить взрывать 
центральную сферу.

Смена режима с 2D на 3D и обратно установит параметры вращения в исходное состояние, ессли они get buggered... полезно, 
если желателен Stereo режим. 
The odds are Stereo mode is not done correctly, might require 2 or 3 Cameras и источники света: 1 для каждого
Red and Blue and 1 for normal mode. 
Might be best done in direct mode поскольку создание двух видов от 2 камер одновременно выглядит странно. 
В каталоге есть исходный си код. Возможно изменение отображения Red/Blue атомов является тем способом, с помощью которого 
можно это сделать... 
Для этого предназначены стереоочки Red/Blue... а не эти on/off lcd goggles.
